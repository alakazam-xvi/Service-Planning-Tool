About the Simulator

Overview

This simulator provides a deterministic model for transit planning. It visualizes how a fleet handles a schedule across varying operating periods, discretizing travel time changes between peak and off-peak conditions.

Route Types

  • Loop Route: Each unit departs from a single terminal and completes a full circuit back. Roundtrip Time (RT) covers the entire loop. Layover is terminal recovery time added on top.
  • Non-Loop Route (A ↔ B): The route has two terminals. All units are dispatched initially from Terminal A. Terminal B's first departure aligns with the moment the first unit is ready to depart from B after queuing. Separate travel times for A→B and B→A can be defined per period, supporting asymmetric traffic conditions.

Scheduling Methods

  • Uniform Trips per Vehicle: Pre-calculates the maximum number of trips that every unit can complete, constrained by the most-restricted unit (the one with the latest start time). All units are then scheduled for exactly that many trips. Trips may start before and end after the operating period ends. Trips extending past the period end are shown as dashed bars on the timetable.
  • Scheduling based on Turn Around Time: Units dispatch as long as their departure falls within the operating window and the full trip completes before the period ends. Results in uneven trip counts across the fleet when the operating window is not a clean multiple of cycle time.

Processes & Formulas

  • Dispatch Interval (Headway):
    • Loop: (RT + Layover) / N
    • Non-Loop: (ttAB + Layover B + ttBA + Layover A) / N
    Period 1 cycle time is used to compute the initial stagger. The summary table shows the implied headway for each period.
  • Non-Loop Offset: Terminal B's first dispatch = Start of Operation + ttAB (Period 1) + Layover B. Subsequent dispatches from both terminals are spaced by the same headway.
  • Trip Assignment: For any departure time, the active period determines the travel time used. Trips spanning a period boundary use the departure period's travel time for the entire leg.

Key Assumptions

  • Homogeneous Fleet: All units are identical and operate at the assumed speeds of their respective periods.
  • Single Crew per Vehicle: No crew rotation modeled. One driver per unit for the full day.
  • Punctuality: Vehicles depart exactly as scheduled. No stochastic delays or bunching.

Limitations

  • Stochasticity: Real-world variance (boarding delays, traffic lights) leading to bus bunching is not modeled.
  • Crew Scheduling: Driver shifts, union constraints, and meal breaks are not mapped against vehicle blocks.
  • Cross-Period Legs: A leg's travel time is determined solely by the period active at departure. No interpolation across period transitions.
